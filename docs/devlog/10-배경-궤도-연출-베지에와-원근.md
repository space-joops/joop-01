# 개발 일지 #10 — 배경 궤도 연출: 베지에 곡선과 원근 착시 🌍🛰️

> 2026-07-21 · 브랜치 `claude/minigame-ambient-orbit`
> 이번 편에서 배우는 것: **2차 베지에 곡선, 도함수로 진행 방향 구하기, ease-in 스케일이 만드는 원근 착시, 장식 레이어의 분리**

9편의 출격 화면은 잘 돌아가지만 한 가지가 아쉬웠습니다 — 파편이 전부
**같은 평면**에서 움직여서 우주라기보다 수족관 같았어요. 그래서 배경에
"멀리 지구 궤도를 도는 위성들"을 흘려보냅니다: 좌하단 지구 수평선 뒤에서
작게 나타나 → 포물선을 그리며 점점 커지다가 → 화면 옆으로 스쳐 나가는,
스타워즈에서 우주선이 카메라 옆을 지나가는 그 문법입니다.

충돌도 점수도 없는 **순수 장식**입니다. 딱 파일 하나(`ambient-orbit.tsx`),
동시 2기, 게임플레이 코드는 한 줄도 안 건드렸어요(마운트 한 줄 제외).

## 1. 포물선은 어떻게 그리나 — 2차 베지에 곡선

"곡선을 따라 움직여라"는 수식 하나면 됩니다:

```
B(t) = (1-t)²·P0 + 2(1-t)t·P1 + t²·P2      (t: 0 → 1)
```

- **P0** 출발점 — 지구 수평선 부근 (화면 좌하단)
- **P2** 도착점 — 화면 밖 좌/우 가장자리
- **P1** 제어점 — 곡선이 "휘어지는 방향". 중상단에 두면 위로 볼록한
  포물선이 됩니다. 곡선은 P1을 **지나가지 않고** P1 쪽으로 끌려갈 뿐이에요.

파이썬으로 치면 `matplotlib`에 점 찍기 전에 매개변수 t로 좌표를 뽑는
파라메트릭 곡선입니다. 매 프레임 `t += dt / travel`로 진행도를 올리고
좌표를 계산해 `translate3d`로 옮기면 끝. CSS 애니메이션 대신 rAF로 직접
계산하는 이유는 회전·크기·투명도를 같은 t에서 한꺼번에 유도하기 위해서예요.

덤: 진행 방향은 **도함수**로 공짜로 얻습니다.

```
B'(t) = 2(1-t)·(P1-P0) + 2t·(P2-P1)
angle = atan2(B'y, B'x)
```

이 각도로 기수를 돌리면 위성이 곡선의 접선 방향을 향해 "날아가는" 것처럼
보입니다. 고등학교 미분이 이렇게 쓰입니다.

## 2. 원근 착시의 핵심 — 일정하게 커지면 가짜다

크기를 `t`에 비례해 키우면 이상하게 밋밋합니다. "커지는 그림"처럼 보여요.
비결은 **가속**입니다:

```ts
scale = 0.12 + (1.1 - 0.12) * Math.pow(t, 1.8); // ease-in
```

실제 원근에서 물체의 화면 크기는 거리에 반비례(1/d)합니다. 등속으로
다가오는 물체는 멀 때는 거의 안 커지다가 가까워지면 **훅** 커져요.
`t^1.8`이 그 곡선을 흉내 냅니다 — 수명의 앞 절반 동안은 깨알만 하다가
마지막 2초에 갑자기 부풀며 화면 밖으로 사라지는 게 "스쳐 지나간다"의
정체입니다. 여기에 두 가지를 더 얹었어요:

- **페이드인** (첫 15%): 투명하게 시작해 스르륵 — "지구 뒤에서 나타났다"
- **상수 필터** `brightness(0.75) saturate(0.8)`: 원경은 살짝 어둡고
  물빠진 색. 단, 필터를 프레임마다 바꾸면 리페인트가 나므로 **한 번만**
  설정합니다 (transform·opacity만 매 프레임 갱신 — 합성 레이어에서 처리).

## 3. 원경은 흔들리지 않는다 — 카메라 문법

이 레이어는 `sortie-field`의 **흔들림 레이어 밖**에 삽니다. 피격 시 화면이
흔들릴 때 근경(파편·펫)만 흔들리고 원경 위성과 지구는 가만히 있어요.
영화 카메라가 그렇습니다 — 셰이크는 피사체 층에서만 일어나고 배경 산맥은
안 흔들리죠. 반대로 배경까지 같이 흔들면 "화면 전체가 스티커 한 장"이라는
게 들통납니다. 레이어를 나눈 순간 공짜로 얻는 깊이감이에요.

같은 이유로 z-order도 게임 레이어 **뒤**: 파편이 위성을 가리고 지나가면
"위성이 더 멀리 있다"는 착시가 강해집니다.

## 4. 장식은 장식답게 — 분리의 규율

`AmbientOrbit`은 9편의 게임 코어와 완전히 분리된 컴포넌트입니다:

- 자체 rAF 루프 + 명령형 `<img>` (9편에서 배운 escape hatch 패턴 재활용)
- `pointer-events: none` — 조그셔틀 입력을 절대 방해하지 않는다
- 상수는 `TUNE`/`DIFFICULTY`가 아닌 자기 파일의 `AMBIENT` 객체 —
  게임성이 아니라 연출이니까
- 게임 쪽 변경은 `action-mode.tsx`에 `<AmbientOrbit />` 마운트 한 줄

파이썬으로 치면 시뮬레이션 모듈에 시각화 코드를 섞지 않고 별도
`viz.py`로 빼는 감각입니다. 나중에 "홈 화면에도 흘려보내자"가 되면
컴포넌트 하나 옮겨 심으면 끝이에요.
