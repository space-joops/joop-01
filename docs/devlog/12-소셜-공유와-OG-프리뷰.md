# 개발 일지 #12 — 소셜 공유: OG 스크랩과 Web Share API 📣

> 2026-07-22 · 브랜치 `claude/social-share`
> 이번 편에서 배우는 것: **오픈 그래프(OG)가 동작하는 원리, Web Share API와 기능 감지, 플랫폼별 공유의 현실(카카오톡·인스타그램), noopener 보안 관례**

마케팅 전략 문서의 결론은 "제로 예산 확산 = 공유가 쉬워야 한다"였습니다.
그런데 지금까지 줍스는 링크를 보내도 **밋밋한 URL 텍스트**만 떴어요. 이번에
관제 설정에 공유 패널을 달고, 링크를 붙여넣는 순간 줍이 카드가 뜨게
만들었습니다.

## 1. OG 미리보기의 원리 — 상대방이 아니라 "서버"가 읽는다

카카오톡에 링크를 붙여넣으면 뜨는 미리보기 카드, 어떻게 만들어질까요?
**카카오(페이스북·디스코드·슬랙도 동일)의 서버가 그 URL을 대신 방문해서
`<head>` 안의 오픈 그래프 태그를 긁어 갑니다**:

```html
<meta property="og:title" content="줍스 (JOOPS) — 우주 쓰레기 청소 위성 펫"/>
<meta property="og:image" content="https://…/og.png"/>  <!-- 1200×630 -->
<meta name="twitter:card" content="summary_large_image"/>
```

Next.js에서는 `layout.tsx`의 `metadata.openGraph` / `metadata.twitter`에
객체로 쓰면 태그가 자동 생성됩니다. 함정 하나: **크롤러는 상대 경로를
못 따라갑니다**. `og:image`가 `/og.png`면 카드가 안 떠요. 그래서
`metadataBase`(우리는 `NEXT_PUBLIC_SITE_URL`)가 필수 — Next.js가 이 기준으로
절대 URL을 만들어 넣습니다.

카드 이미지는 **정적 `public/og.png`(1200×630)** 를 택했습니다. Next에는
런타임에 이미지를 그려주는 `ImageResponse`도 있지만, 우리 카드는 내용이
변하지 않으니 미리 구워 두는 쪽이 빠르고 확실합니다. 소스는
`scripts/og-image.svg` — 에셋 팩 스타일로 그린 SVG를 크로뮴 스크린샷으로
래스터화했고, 소스를 레포에 남겨 언제든 다시 구울 수 있습니다.

## 2. Web Share API — 버튼 하나로 모든 앱

공유의 웹 표준은 **Web Share API** 하나입니다:

```ts
await navigator.share({ title, text, url }); // OS 공유 시트가 뜬다
```

모바일에서 이 한 줄이면 카카오톡·인스타그램 DM·문자·메일 등 **설치된 모든
앱**이 시트에 나옵니다. 우리가 앱별 연동을 안 해도 OS가 다 해주는 거예요.
단, 데스크톱 브라우저엔 없는 경우가 많아 **기능 감지**가 필수:

```ts
typeof navigator.share === "function"  // 파이썬의 hasattr(obj, "share") 감각
```

지원하면 대표 버튼으로 노출, 아니면 숨기고 아래 수단들이 대신합니다.
"있으면 최신 기능, 없으면 폴백" — 점진적 향상(progressive enhancement)이라는
웹의 기본 예의입니다.

## 3. 플랫폼별 현실 — 전부 같은 방법이 통하지 않는다

| 플랫폼 | 방법 | 비고 |
| --- | --- | --- |
| 페이스북 | `facebook.com/sharer/sharer.php?u=…` | 공식 공유 URL, 팝업 |
| X | `x.com/intent/post?url=…&text=…` | 문구까지 미리 채움 |
| 카카오톡 | JS SDK `Share.sendScrap` (앱 키 필요) | 키 없으면 **링크 복사 + 안내** — 붙여넣기만 해도 OG 카드가 뜬다 |
| 인스타그램 | **웹 공유 URL이 없음** | 시스템 공유 시트 또는 링크 복사로 정직하게 안내 |

인스타그램처럼 웹 진입점 자체가 없는 플랫폼은 흉내 내지 않고 한계를 UI
문구로 알려주는 게 낫습니다. 카카오 SDK는 첫 탭에서만 동적 로드해서
(`<script>` 주입) 평소 페이지 성능엔 영향이 없어요. 참고로 SDK의
`sendScrap`도 결국 **우리 OG 태그를 스크랩**해 카드를 만듭니다 — 1장에서
만든 기반이 모든 경로에 재사용되는 구조.

## 4. 작은 보안 관례 — noopener

공유 팝업은 전부 이렇게 엽니다:

```ts
window.open(url, "_blank", "noopener,noreferrer,…");
```

`noopener`가 없으면 새 창이 `window.opener`로 **우리 페이지를 조작**할 수
있습니다(탭내빙 피싱). 요즘 브라우저는 기본값이 안전해졌지만 명시가
표준 관례예요. 링크 복사는 Clipboard API(`navigator.clipboard.writeText`)
+ 실패 시 주소 노출 폴백으로 처리했습니다.
