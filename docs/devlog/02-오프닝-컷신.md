## 1. Framer Motion — "상태를 선언하면 애니메이션은 따라온다"

지난 편의 CSS `@keyframes`는 "이 애니메이션을 무한 반복해라"였다면, Framer Motion은
**"지금 상태가 이렇다"고 선언하면 이전 상태에서 지금 상태로 가는 길을 라이브러리가
알아서 그려주는** 방식입니다.

```tsx
<motion.div
  animate={{ scale: revealed ? 1.45 : 1 }}   // 상태 선언
  transition={{ duration: 3, ease: "easeInOut" }}  // 가는 방법
/>
```

`revealed`가 `false → true`로 바뀌는 순간, 줌인이 시작됩니다. 파이썬 감각으로는
"명령형 `for` 루프로 프레임을 그리는 대신, 선언형으로 목표만 주는" 차이예요.
React가 UI를 다루는 철학("상태 → 화면")을 애니메이션에도 그대로 적용한 겁니다.

배열을 주면 **키프레임**이 됩니다. 출격 직전의 설레는 떨림이 이렇게 한 줄입니다:

```tsx
animate={{ x: [0, -5, 5, -4, 4, 0] }}   // 부르르
```

## 2. AnimatePresence — 사라지는 것들의 애니메이션

React에서 컴포넌트가 조건부 렌더링으로 사라지면 **즉시 DOM에서 제거**됩니다.
퇴장 애니메이션을 보여줄 틈이 없죠. `AnimatePresence`가 이 문제를 풀어줍니다:

```tsx
<AnimatePresence mode="wait">
  {scene === 1 && <SceneLostSky key="scene-1" ... />}
  {scene === 2 && <SceneOperatorRoom key="scene-2" ... />}
</AnimatePresence>
```

- 자식이 사라질 때 바로 제거하지 않고, `exit={{ opacity: 0 }}` 애니메이션을 **끝까지
  재생한 뒤** 제거합니다.
- `mode="wait"`: 이전 장면의 퇴장이 끝나기를 기다렸다가 다음 장면을 등장시킴 →
  자연스러운 크로스페이드. (파이썬 비유: 컨텍스트 매니저의 `__exit__`가 완료된 후
  다음 블록에 진입하는 느낌)
- `key`가 필수입니다. React는 key로 "누가 사라졌는지"를 추적하니까요.

## 3. 타임라인 연출 패턴 — setTimeout 오케스트라

컷신은 "0.7초에 나레이션, 4초에 반전, 10.5초에 다음 장면"처럼 **시간표**가 필요합니다.
각 장면이 쓴 패턴:

```tsx
useEffect(() => {
  const timers = [
    setTimeout(() => setStep(1), 700),      // 나레이션 1줄
    setTimeout(() => setRevealed(true), 4000), // 반전!
    setTimeout(() => onDoneRef.current(), 10500), // 자동 진행
  ];
  return () => timers.forEach(clearTimeout);  // 청소는 필수
}, []);
```

배울 포인트 두 가지:

1. **cleanup 함수** — useEffect가 반환하는 함수는 컴포넌트가 사라질 때 실행됩니다
   (컨텍스트 매니저의 `__exit__`). 유저가 '건너뛰기'를 눌러 장면이 중간에 언마운트돼도
   타이머가 유령처럼 남아 다음 장면을 강제로 넘기는 버그를 막아줍니다.
2. **`onDoneRef` 패턴** — 부모가 준 콜백을 ref에 담아 쓰면, 콜백 함수가 매 렌더마다
   새로 만들어져도 타이머를 다시 걸 필요가 없습니다. (의존성 배열 문제의 고전적 해법)

## 4. 반전 연출의 트릭 — 별과 파편은 같은 자리에 겹쳐 있다

1장의 핵심: "아름다운 별빛인 줄 알았던 것이 사실 금속 파편이었다."

구현은 의외로 단순합니다. 같은 좌표에 **별 SVG와 파편 SVG를 포개 놓고**, `revealed`
상태에 따라 opacity를 서로 교차시킵니다. 파편은 드러나면서 살짝 회전(`rotate`)해
"조명이 돌아가며 정체가 보이는" 느낌을 더했어요. 하늘 배경색도 꿈결 같은 남보라에서
차가운 검정으로 함께 식습니다(`animate={{ backgroundColor }}`).

배치 좌표는 1편에서 배운 **인덱스 기반 의사 랜덤**을 그대로 썼습니다. 다면체 지구
(`faceted-earth.tsx`)도 삼각형 좌표를 모듈 로드 시점에 한 번만 계산하는 결정적 코드라
hydration 걱정이 없습니다.

## 5. 3장은 왜 "보는" 컷신이 아니라 "만지는" 컷신인가

시나리오 문서의 핵심 요구: *유저가 직접 화면을 터치해 위성을 쓰다듬어야* 하고, 그 교감이
튜토리얼 전의 정서적 유대가 되어야 합니다. 온보딩 원칙(90초, 텍스트 설명 금지)에 따라
**설명 문구 없이 손가락 이모지가 쓰다듬는 궤적을 그리며 힌트**를 줍니다.

- 1편에서 만든 포인터 추적 패턴 재사용: 누적 이동 30px마다 교감 1회(💛 파티클 + 함박웃음).
- 콕 찍기만 해도 교감 1회로 인정 — 온보딩에서는 실패가 없어야 하니까요.
- 8회 교감이 쌓이면: 떨림(설렘) → 추진기 점화 → 발진(별빛 줄기가 스치는 속도감) →
  타이틀 로고 → "관제 콘솔 접속" 버튼.
- 발진할 때 지구가 아래로 밀려나는 **패럴랙스**로 상승감을 만들었습니다.

## 6. localStorage 분기와 hydration — "서버는 localStorage를 모른다"

컷신은 첫 방문에만 보여야 합니다. 봤다는 기록은 `localStorage`에 남기는데, 여기 함정이
하나 있습니다. **localStorage는 브라우저에만 존재**해서, 서버 렌더링 중에 읽으려 하면
터집니다. 렌더 함수 안에서 읽으면 서버/클라이언트 결과가 어긋나 hydration 오류가 나고요.

해법은 `game-root.tsx`의 3단계 분기입니다:

```
서버가 그리는 첫 화면: 무조건 "boot" (검은 화면)
        ↓ 마운트 후 useEffect (브라우저에서만 실행)
localStorage 확인 → "intro" 또는 "game"
```

첫 화면이 잠깐 검은 것은 오히려 연출입니다 — 컷신 1장이 암전에서 시작하니까요.
이 기록은 나중에 Supabase 프로필로 옮겨 기기가 바뀌어도 유지되게 할 예정입니다.

## 7. 소소하지만 알아두면 좋은 것들

- **타자기 효과** (2장): `setInterval`로 70ms마다 글자 수 state를 +1 하고
  `LOGIN_TEXT.slice(0, typedCount)`로 자릅니다. 커서 `▌`는 Tailwind `animate-pulse`.
- **motion 요소와 Tailwind transform 클래스는 섞지 말 것** — `-translate-x-1/2` 같은
  클래스와 framer의 `animate`는 둘 다 `transform`을 건드려 서로 덮어씁니다. 그래서
  가운데 정렬은 부모 flex로 처리했습니다. (한 번 밟아본 사람이 알려주는 지뢰)
- **SVG 부분 애니메이션** — 아기 위성의 안테나만 파닥이게 하려고 `<g>`에 CSS 클래스를
  주고 `transform-box: fill-box`로 회전축을 안테나 뿌리에 고정했습니다.
- **번들 크기 트레이드오프** — Framer Motion으로 First Load JS가 117→164kB가 됐습니다.
  아직 허용 범위지만, 나중에 `LazyMotion`으로 필요한 기능만 로드하는 최적화 여지를
  기록해 둡니다.
- **소리는 아직 없다** — "기분 좋은 소리"는 모바일 브라우저의 자동재생 정책(유저
  제스처 전 재생 불가) 때문에 사운드 시스템을 설계할 때 한꺼번에 다룰 예정입니다.
