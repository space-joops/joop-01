# 개발 일지 #11 — 궤도 관제 화면: 지상 자취 수학과 정사영 투영 🌍📡

> 2026-07-21 · 브랜치 `claude/orbit-monitor`
> 이번 편에서 배우는 것: **케플러 제3법칙에서 수치 유도하기, 지상 자취(ground track)가 사인 곡선인 이유, 두 가지 지도 투영, 24시간 패스 스캔, "하나의 데이터·여러 투영" 설계**

줍스는 "궤도를 돌며 파편을 줍는 위성" 게임인데, 정작 줍이가 **지구 어디
위를 날고 있는지** 보여주는 화면이 없었습니다. 이번에 홈 헤더의 📡 버튼으로
여는 **궤도 관제 화면**을 만들었어요 — 세계지도 위 자취, 지구본, 텔레메트리,
기지국 패스 예측까지. 진짜 위성 추적 사이트의 축소판입니다.

## 1. 숫자를 정하지 않고 유도한다 — 케플러 제3법칙

고도 420km(ISS와 같은 저궤도)만 정하면 나머지 수치는 물리가 정해줍니다:

```ts
const ORBIT_A = 6371 + 420;                          // 궤도 반경 a (km)
const PERIOD_S = 2 * Math.PI * Math.sqrt(a ** 3 / μ); // ≈ 92.8분
const ORBIT_VEL = Math.sqrt(μ / a);                   // ≈ 7.66 km/s
```

μ(뮤)는 지구 중력 상수 398,600 km³/s². 텔레메트리에 뜨는 92.8min과
7.66km/s는 하드코딩이 아니라 **케플러 제3법칙(T² ∝ a³)의 계산 결과**예요.
고도를 550km로 바꾸면 주기·속도가 알아서 따라 바뀝니다. 파이썬으로 치면
매직 넘버 대신 `dataclass`의 `@property`로 유도 값을 만드는 감각.

## 2. 지상 자취는 왜 사인 곡선인가

위성은 기울어진 평면 위의 원을 돌 뿐입니다. 궤도 위상 θ에서 직하점은:

```ts
lat = asin(sin(경사각) · sin θ)          // 위도는 ±경사각 사이를 진동
lon = atan2(cos(경사각)·sinθ, cosθ)      // 관성계 경도
    + 시작경도 − 지구자전(360° × t/86164s) // ← 이게 핵심!
```

위도가 sin으로 진동하니 세로로 물결치고, **지구가 밑에서 자전하며 경도를
서쪽으로 밀어내니** 물결이 매 바퀴 22.9°씩 왼쪽으로 이동합니다. 위성 추적
사이트에서 보던 그 사인 곡선이 이 두 줄에서 나옵니다. 86164초는 하루가
아니라 **항성일**(별 기준 자전) — 태양일 86400초를 쓰면 미세하게 어긋나요.

## 3. 투영 두 가지 — 데이터는 위경도로, 그림은 각자

대륙 실루엣은 `[경도, 위도]` 폴리곤 13개(수제 간략화)로 한 파일에 두고,
**두 화면이 서로 다른 투영으로 같은 데이터를 그립니다**:

- **평면 지도(등장방형)**: 투영이랄 것도 없이 `x = 경도+180, y = 90−위도`.
  viewBox를 360×180으로 잡으면 위경도가 곧 SVG 좌표입니다. 유일한 함정은
  **날짜변경선** — 경도가 +179°→−179°로 점프하면 선이 화면을 가로지르므로
  |Δx| > 180일 때 폴리라인을 끊습니다.
- **지구본(정사영)**: "아주 멀리서 본 지구". 직하점(φ₀, λ₀)을 중앙에 두면

  ```
  x = cosφ·sin(λ−λ₀)
  y = cosφ₀·sinφ − sinφ₀·cosφ·cos(λ−λ₀)
  보임 = sinφ₀·sinφ + cosφ₀·cosφ·cos(λ−λ₀) > 0
  ```

  뒷반구 정점은 버리는 대신 **원판 가장자리(림)로 밀어붙입니다** —
  (x,y)를 정규화해 길이 1로 만드는 한 줄. 이러면 반쯤 걸친 대륙이
  찢기지 않고 실루엣이 닫혀요. 지구본이 직하점을 따라 돌기 때문에
  줍이 마커는 항상 정중앙 — 카메라가 위성을 따라가는 연출입니다.

## 4. 패스 예측 — 우아한 해석해 대신 무식한 스캔

"기지국 상공을 언제 지나가나"를 수식으로 푸는 건 꽤 어렵지만, 게임에는
**30초 간격으로 24시간을 훑는 스캔**이면 충분합니다 (2,880번 — 1ms도 안
걸려요). 파이썬의 `np.arange`로 시각 배열 만들어 조건 마스크 씌우는 것과
같은 발상:

```ts
for (let t = now; t <= now + 24h; t += 30s) {
  if (앙각(t) >= 10°) 패스 구간에 누적;   // 시작·지속·최대앙각 기록
}
```

앙각은 구면 기하 한 줄: 직하점과 기지국의 중심각 ψ에서
`elev = atan2(cosψ − R/a, sinψ)`. 링크 상태(LINK OK / NO SIGNAL),
커버리지 원(앙각 10° 등고선), ★ 특급 패스(최대 45°↑)가 전부 여기서
파생됩니다.

## 5. 설계 노트 — 시계 하나, 순수 함수 여럿

- 화면의 유일한 상태는 `now`(500ms 갱신) 하나. 네 패널은 각자
  `subpointAt(now)` 같은 **순수 함수**를 불러 필요한 값을 계산합니다.
  스토어·서버와 완전 무관한 읽기 전용 화면이라 게임에 영향이 없어요.
- 무거운 것(자취 폴리라인·24h 스캔)은 `useMemo`로 5초~10분 단위 캐시.
- 궤도는 **고정 epoch 기준 결정론** — 언제 접속해도 REV 카운터와 위치가
  이어져 보입니다. 보상 계산이 아니므로 클라이언트 시계로 충분합니다
  (원칙 4는 보상에만 해당).
- 관제 용어(LAT/LON/ELEV)는 영문 그대로 — "오퍼레이터의 콘솔"이라는
  세계관 연출(디제틱 UI)입니다. 안내 문구는 전부 한국어로.
