# 개발 일지 #16 — 부유 파편과 수집함: 방치와 액션 사이의 간식 🎒

> 2026-07-22 · 브랜치 `claude/inventory-ambient`
> 이번 편에서 배우는 것: **미니 수집 루프의 밸런스 설계, 로컬 전용 통계의 경계, 도감 UI의 "?" 심리, 등장·퇴장의 대칭, Playwright와 움직이는 요소**

## 1. 홈에 간식을 흘려보내다 — 부유 파편

홈 화면은 그동안 "보는" 화면이었습니다. 펫을 탭/쓰다듬는 것 말고는 할 게
없었죠. 이제 미니게임의 우주 쓰레기가 가끔(동시 최대 2개, 평균 ~23초에
하나) 홈에도 흘러들어옵니다. 탭하면 줍이에게 빨려가며 냠.

밸런스가 새지 않게 두 가지 원칙을 세웠어요:

- **보상은 기존 규칙 재사용**: 탭 수집 = `eatDebris()` 그대로 (파편+1·
  EXP+2·데이터 축적·배터리-2). 새 보상 경로를 만들면 밸런스 구멍이 되니,
  기존 수도꼭지에 연결만 했습니다.
- **주 획득처는 미니게임**: 홈은 어디까지나 간식. 동시 2개 상한과 낮은
  스폰 확률이 이를 강제합니다. (동면·절전엔 스폰 안 함 — 어차피 먹지도
  못하니까)

## 2. 수집함 — "???"가 일하게 하라

푸터가 4버튼(☀️ 충전 · 📡 전송 · 🎒 수집함 · 🔧 강화, 전부 같은 크기)이
됐고, 수집함을 열면 파편 8종 도감이 나옵니다. 핵심 디테일은 **아직 못
주운 종류를 숨기지 않고 흐린 실루엣 + ?로 보여주는 것** — "빈 칸"은
채우고 싶어지는 심리를 자극합니다. 도감류 게임(포켓몬 도감이 원조)의
검증된 장치예요.

미니게임도 이제 종류별로 셉니다. `SortieResult`에 `kindCounts`를 추가해
귀환 정산 때 `recordSortie()`로 도감에 합산 — 출격 횟수·충돌 횟수도 함께
쌓입니다.

## 3. 로컬 전용 통계 — 서버를 안 건드리는 선택과 그 경계

도감·출격·충돌 통계는 **서버 스키마(joop_01_*)에 컬럼이 없습니다**.
persist(localStorage)에만 저장되는 로컬 전용이에요. 왜 이렇게 했나:

- 이 통계는 보상·밸런스에 영향이 없는 "재미 데이터"라 치트 걱정이 없다
- 마이그레이션 + RPC 확장 없이 순수 프론트 작업으로 완결된다

대신 경계도 명확합니다: **기기를 바꾸면 도감이 초기화**됩니다(게임 코어
상태는 서버에 있으니 무사). 도감이 소셜 자랑거리가 되는 시점이 오면
서버로 승격해야 해요 — 그때가 오면 마이그레이션 파일과 `joop_01_sync_pet`
확장이 필요하다고 스토어 주석에 박아 뒀습니다. 파이썬으로 치면 "캐시에
넣을 것과 DB에 넣을 것"의 구분 감각입니다.

## 4. 플라이바이 다듬기 — 등장과 퇴장의 대칭

스타링크형 위성(납작한 플랫 패널 버스 + 대형 단일 태양돛)이 함대에 합류해
6종이 됐습니다. 그리고 퇴장 버그 아닌 버그 하나: 기체가 경로 끝(t=1)에
닿으면 노드를 **즉시 제거**했는데, 화면 경계 근처라면 "깜빡" 하고 사라져
보였어요. 등장은 페이드인이 있는데 퇴장은 없었던 거죠.

```ts
opacity = max × min(1, t/fadeIn) × min(1, (1−t)/fadeOut)
```

곱셈 하나로 등장·퇴장이 대칭이 됐습니다. 연출의 기본기: **나타나는
방식과 사라지는 방식은 짝을 이뤄야** 눈이 편안합니다.

## 5. 오늘의 테스트 함정 — 움직이는 건 탭할 수 없다?

Playwright의 `tap()`은 요소가 "안정될 때까지"(bounding box가 멈출 때까지)
기다립니다. 그런데 부유 파편은 영원히 떠다니죠 — 테스트가 무한 대기에
빠졌습니다. 실사용자는 아무 문제 없이 탭할 수 있는데요.

해결: 테스트에서 `dispatchEvent("pointerdown")`으로 이벤트를 직접
쏘거나 `{ force: true }`를 씁니다. **"테스트가 실패했다 ≠ 제품이
잘못됐다"** — 도구의 가정(요소는 결국 멈춘다)이 게임 UI(계속 움직인다)와
충돌한 경우입니다.
