## 1. "재택근무 책상"을 버린 이유 — 모바일 디제시스

원래 오프닝 2장은 어질러진 방, 책상 위 데스크톱 모니터에 "재택근무
오퍼레이터 접속 완료"가 타자기처럼 찍히는 장면이었습니다. 분위기는
있었지만 이상한 점이 하나 있어요 — **줍스는 폰으로 하는 게임**인데, 화면
속 주인공은 데스크톱 앞에 앉아 있었죠.

그래서 프레이밍을 뒤집었습니다. 어둠 속에서 스마트폰이 켜지고, 메신저로
클리어 스카이의 환영 인사가 도착합니다:

> "장비는 필요 없어요 — 지금 들고 계신 그 폰이 관제 콘솔이거든요 📱"

이 대사가 이번 개편의 심장입니다. **게임을 플레이하는 바로 그 기기가
세계관 속 장비가 되는 것** — 연출 용어로 '디제시스(diegesis, 작품 세계
내부의 것)'라고 합니다. 유저가 폰을 드는 행위 자체가 "관제 콘솔 접속"이
되므로, 화면 밖 현실과 화면 안 이야기의 경계가 사라져요. UI도 익숙한
메신저 문법(타이핑 인디케이터 ●●●, 말풍선, 영상 카드)이라 별도 설명 없이
읽힙니다 — 온보딩 원칙(텍스트 최소화)과도 맞습니다.

1장도 손봤습니다. "반짝임은 별빛이 아니다"라는 반전만 있고 **왜** 하늘이
그렇게 됐는지가 없었어요. 나레이션을 3박자로 늘려 배경 스토리 문서의
캐논을 심었습니다: 버려진 위성들의 연쇄 충돌(케슬러 신드롬) → 하늘을 가둔
파편 띠 → 사람들이 부르는 이름 **'대폐색'**. 반전 순간엔 "— 궤도 대폐색
이후 7년 —" 연대기 캡션이 떠서 지금이 어느 시대인지 못박습니다.

## 2. 팝인의 정체 — Suspense fallback={null}

홈 화면의 3D 줍이는 이렇게 로드됩니다:

```tsx
<Suspense fallback={null}>
  <PetModel url={url} />   {/* useGLTF(url)가 로드 완료까지 suspend */}
</Suspense>
```

`useGLTF`는 React Suspense와 붙어 있어서, GLB 다운로드·파싱이 끝날 때까지
컴포넌트 렌더를 "정지"시킵니다. 문제는 `fallback={null}` — 정지된 동안
**아무것도 안 보이다가**, 로드가 끝나는 순간 완성된 모델이 **툭** 나타나는
거예요(팝인). 파이썬으로 치면 `await` 중에 스피너도 없이 빈 터미널을
보여주는 셈입니다.

## 3. 해결 두 겹 — 폴백 크로스페이드 + 착륙 연출

**① 2D 폴백 크로스페이드.** 로드되는 동안 같은 캐릭터의 2D 스티커
(9편에서 배치한 SVG 팩의 `petSprite()` 재사용!)가 자리를 지킵니다.
모델이 준비되면 `onReady` 콜백 → 스티커가 0.5초 페이드 아웃:

```tsx
function PetModel({ onReady }) {
  const { scene } = useGLTF(url);      // 여기를 지났다 = 로드 완료
  useEffect(() => onReady?.(), [onReady]);
```

Suspense의 성질을 그대로 이용한 트릭입니다 — suspend가 풀려 useEffect가
실행되는 시점이 곧 "준비 완료" 신호예요. 진화로 모델이 바뀔 때도 새 단계의
2D 스티커가 이어받으므로 어떤 전환에도 빈 화면이 없습니다.

**② 착륙 연출.** 준비된 모델도 그냥 나타나면 심심하니, 위에서 스르륵
내려앉으며 커지는 0.9초를 줬습니다:

```ts
st.entrance = Math.min(1, st.entrance + dt / 0.9);
const arrive = 1 - Math.pow(1 - st.entrance, 3); // ease-out cubic
root.scale.setScalar(rig.scale * (0.55 + 0.45 * arrive));
root.position.y = (1 - arrive) * 0.4;
```

`1-(1-t)³`은 처음엔 빠르고 끝에서 부드럽게 멈추는 ease-out 곡선 —
"도착"의 느낌을 만드는 표준 이징입니다. 서사적으로도 딱 맞아요: 인트로
3장에서 하늘로 발진한 줍이가, 홈 화면(관제 콘솔)에 **도착**하는 겁니다.
컷신과 게임 화면이 한 호흡으로 이어집니다.

## 4. 배울 포인트 정리

- **디제시스 프레이밍**: 플랫폼(모바일)과 세계관을 일치시키면 몰입은
  공짜로 따라온다. UI 문법도 유저가 이미 아는 것(메신저)을 빌릴 것.
- **Suspense 폴백은 UX다**: `fallback={null}`은 "로딩을 안 보여주겠다"는
  선언이다. 자리를 지킬 대체물(스켈레톤·스프라이트)을 항상 고민할 것.
- **로드 완료 신호는 공짜**: suspend 아래 컴포넌트의 첫 useEffect가 곧
  "준비 완료"다. 별도 이벤트 배선이 필요 없다.
- **등장은 연출 기회**: 팝인을 없애는 김에 서사(발진→착륙)를 이어붙이면
  기술 수정이 스토리텔링이 된다.
