> 2026-07-21 · 여덟 번째 기능 구현 · 브랜치 `1` · v0.5.0
> 이번 편에서 배우는 것: **범용 인벤토리 스키마(kv 패턴), 서버 경제 검증, upsert(on conflict), 게임 루프 닫기**

6편에서 EXP의 소비처(진화)를 만들었으니, 이번엔 **파편의 소비처**입니다.
이제 줍스의 코어 루프가 완전히 닫힙니다:

```
청소(탭·출격·방치) → 파편 획득 → 강화 구매 → 더 잘 청소 → …
```

## 1. 강화 3종 — 기획서 4.1 그대로

| 강화 | 효과 | 체감 |
| --- | --- | --- |
| 📦 화물칸 | 파편당 데이터 축적 ÷(1+0.25×Lv) | 전송 없이 더 오래 수집 |
| 🤖 AI 코어 | 오프라인 수집 +20%/Lv | 자고 일어나면 더 쌓여 있음 |
| ☀️ 태양광 패널 | 충전량 +25%/Lv | 충전 버튼 한 번이 더 든든 |

비용 곡선은 50 → 90 → 160 → 290 → 520 (최대 Lv.5). 지수 곡선(×~1.8)이라
첫 강화는 금방, 만렙은 각오가 필요합니다 — 방치형 게임의 표준 리듬.

## 2. inventory — "컬럼 추가"의 유혹을 이긴 kv 패턴

업그레이드 레벨을 `pets`에 컬럼 3개로 추가할 수도 있었습니다. 대신 개발
가이드가 요구한 **범용 inventory 테이블**로 갔습니다:

```sql
create table joop_01_inventory (
  user_id uuid, item_key text, qty int,
  unique (user_id, item_key)
);
-- 업그레이드는 item_key='upgrade_cargo', qty=레벨
```

이유: 기획서에는 쓰레기 부품, 특수 코어, 데칼(스킨)이 대기 중입니다.
새 아이템 종류가 생길 때마다 `alter table`을 반복하는 대신, **행 하나 추가**로
끝나는 구조. 파이썬으로 치면 클래스에 속성을 계속 늘리는 대신
`dict[str, int]` 하나를 두는 선택입니다. (물론 만능은 아닙니다 — 아이템별
복잡한 속성이 생기면 그때 테이블을 분리하면 됩니다.)

구매의 레벨 갱신은 **upsert**로:

```sql
insert into joop_01_inventory (user_id, item_key, qty)
  values (...)
  on conflict (user_id, item_key)
  do update set qty = excluded.qty;
```

"없으면 insert, 있으면 update"를 한 문장으로 — 경쟁 조건 걱정도 DB가 맡아줍니다.

## 3. 경제는 서버가 지킨다 (이제 세 번째라 익숙하죠?)

정산(4편) → 승급(6편) → 이번엔 **구매**. 패턴은 동일합니다:

- `joop_01_buy_upgrade()`가 펫 행을 잠그고(`for update`) 잔액 검증 → 차감 →
  레벨 업을 한 트랜잭션에서 처리. 클라이언트가 "나 파편 999개야"라고 우겨도
  서버 장부가 진실.
- 정산 함수 v2가 인벤토리를 읽어 AI 코어·화물칸을 오프라인 수집에 반영 —
  실측: AI 코어 Lv.2로 5시간 방치 시 30개 대신 정확히 42개(+40%).
- E2E 7종: 잔액 부족 → 구매 → 연속 구매 → 재부족 → 엉뚱한 키 거절 →
  정산 반영 → RLS 조회. 전부 통과.

클라이언트 상수(`UPGRADE_COSTS`)는 늘 그렇듯 **표시용 미러**입니다.

## 4. 클라이언트 — 효과가 세 군데에 스며든다

업그레이드 레벨은 부팅 때 `bootPet()`이 인벤토리와 함께 싣고 옵니다.
효과 적용은 스토어의 세 지점:

- `chargeSolar()` — 충전량 `solarChargeAmount(lv)`
- `eatDebris()` / `finishSortie()` — 데이터 축적에 `cargoDataFactor(lv)`
- 오프라인 수집은 서버 정산 함수가 알아서 (클라이언트 관여 없음)

로컬 모드 폴백(`buyUpgradeLocal`)도 같은 비용표를 씁니다 — 6편의
`configured: false` 신호 패턴 재사용.

## 5. UI — 파편 카운터 옆의 작은 공방

푸터가 3버튼이 됐습니다: ☀️ 충전 / **🔧 강화** / 📡 전송. 강화 시트는
아래에서 올라오는 바텀 시트로, 카드마다 레벨 핍(●●○○○)·현재 효과·다음
비용을 보여줍니다. 잔액이 모자라면 버튼이 스스로 잠깁니다.

## 6. 남긴 것

- 태양광 Lv는 서버 정산에 영향이 없습니다(충전은 온라인 행위) — 나중에
  "오프라인 배터리 지속 +%"로 확장하면 방치 시간이 길어지는 재미가 생깁니다.
- 부품·데칼 등 비강화 아이템이 들어오면 inventory에 그대로 얹습니다.
- 구매 직후 30초 동기화와의 미세한 경합(오래된 스냅샷이 서버 잔액을 덮는
  창)은 이론상 존재 — 다음 치트 방지 편에서 sync 검증과 함께 다룹니다.
