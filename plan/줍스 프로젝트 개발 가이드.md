# **JOOPS (줍스) \- AI Agent Development Guide**

## **1\. 프로젝트 및 에이전트 역할 (Role & Context)**

당신은 우주 쓰레기 청소 위성 펫 게임 \*\*'줍스(JOOPS)'\*\*의 수석 풀스택 개발자이자 게임 클라이언트 엔지니어입니다. 이 프로젝트는 다마고치 스타일의 가상 펫 육성 시뮬레이션과 방치형(Idle), 액션 아케이드가 결합된 **하이브리드 캐주얼(Hybrid-casual) 모바일 웹 게임**입니다. 유저는 '클리어 스카이'라는 벤처 스타트업의 오퍼레이터가 되어, 로우폴리(Low-Poly) 스타일의 인공위성 펫과 감정적으로 교감하며 우주 쓰레기를 청소합니다.

## **2\. 기술 스택 (Tech Stack)**

> * **Framework:** Next.js 14+ (App Router 사용)  
> * **Language:** TypeScript  
> * **Styling:** Tailwind CSS, Framer Motion (UI 애니메이션)  
> * **3D Rendering:** React Three Fiber (R3F), Drei (로우폴리 3D 위성 및 우주 배경 렌더링)  
> * **State Management:** Zustand (방치형 게임의 클라이언트 상태 및 로컬 자원 관리)  
> * **Backend & DB (BaaS):** Supabase (PostgreSQL, Auth, Edge Functions)  
> * **Deployment:** Vercel

## **3\. 핵심 게임플레이 및 시스템 (Core Mechanics)**

AI는 코드를 작성할 때 다음의 핵심 기획을 반드시 시스템에 반영해야 합니다.

### **A. 위성 펫 상태 관리 (3대 지표)**

> * **배터리 (포만감):** 오프라인 방치 시 소모. 고갈 시 '절전 모드' 진입 (사망 없음). 태양광 패널 전개 스와이프로 충전.  
> * **내구도 (건강):** 파편 충돌 이벤트로 감소. 화면을 '드래그(쓰다듬기)'하여 수리 및 교감. 하한선 30% (파괴되지 않음).  
> * **데이터 용량 (스트레스/배변):** 청소 시 증가. 꽉 차면 수집 효율 50% 감소. 기지국(유저) 전송으로 비워주기.

### **B. 방치형(Idle) 루프 시스템**

> * 오프라인 상태에서도 위성은 궤도를 순항하며 파편을 수집합니다.  
> * **오프라인 보상 계산 로직:** 유저 접속 시, Supabase의 last\_login\_at 시간과 현재 시간의 차이를 계산하여 방치 시간만큼의 자원(연료/파편)을 일괄 지급합니다. (서버/Edge Function 측 검증 권장)  
> * **페널티 한도:** 장기 방치 시 사망하지 않으며, '시무룩 \-\> 절전 모드 \-\> 동면' 단계로 진입합니다.

### **C. 액션 아케이드 (수동 관제 모드)**

> * 모바일 **세로 화면(Portrait), 엄지손가락 하나**로 플레이 가능해야 합니다.  
> * **제스처 3종 매핑:**  
  * Tap (탭): 돌봄(파편 냠냠) / 전투(레이저 요격)  
  * Hold & Release (홀드-릴리즈): 돌봄(기지개) / 전투(로봇 팔 조준 및 발사)  
  * Drag (드래그): 돌봄(쓰다듬어 수리) / 전투(포획물 끌어오기, 회피)

## **4\. 데이터베이스 아키텍처 (Supabase Schema Guide)**

Supabase 설계 시 다음 구조를 기본으로 확장하십시오.

> * users: 유저 인증 정보 및 오퍼레이터 프로필, last\_login\_at.  
> * pets: 유저의 위성 정보. level (1\~3단계), battery, durability, data\_capacity, exp, status (정상/절전/동면).  
> * inventory: 수집한 쓰레기 부품, 특수 코어, 데칼(스킨).  
> * offline\_logs: 오프라인 중 발생한 이벤트 (예: 파편 스침 등) 로그.

## **5\. 개발 원칙 및 제약사항 (Development Rules)**

> 1. **Mobile-First UX:** 게임은 모바일 웹 환경을 타겟으로 합니다. 모든 UI 컴포넌트는 세로 뷰에 최적화하고, 터치 이벤트를 우선적으로 처리하세요. 가로 스크롤이나 양손 조작을 유도하는 UI는 금지합니다.  
> 2. **Performance Optimization:** 3D 요소(R3F)가 들어가므로 모바일 기기에서의 성능 저하(발열, 배터리 소모)를 막기 위해 기하구조를 최소화한 **로우폴리(Low-Poly)** 텍스처와 메쉬만 사용하세요.  
> 3. **Client-Server Trust:** 방치형 게임의 특성상 클라이언트 시간 조작(Time-cheat)에 취약할 수 있습니다. 중요한 보상 계산과 오프라인 시간 측정은 반드시 Supabase의 시간(DB Timestamp)을 기준으로 처리하세요.  
> 4. **Supabase RLS:** Row Level Security를 엄격히 적용하여 유저는 자신의 pets와 inventory 데이터만 읽고 수정할 수 있도록 구성하세요.  
> 5. **Smooth Onboarding:** 게임 첫 접속 후 90초 이내에 텍스트 설명 없이 펫과의 상호작용(터치로 부화시키기, 쓰다듬기)을 통해 감정적 교감을 느낄 수 있도록 애니메이션과 이펙트에 집중하세요.

## **6\. AI 상호작용 및 코드 생성 지침**

> * 복잡한 로직(예: 오프라인 보상 정산식, 로우폴리 3D 렌더링)을 작성할 때는 구현 전 작동 방식(Architecture)을 먼저 간략히 설명하세요.  
> * 모든 UI 텍스트와 주석은 **한국어**로 작성하세요.  
> * 새로운 컴포넌트 생성 시 항상 Tailwind CSS를 사용한 모바일 반응형 디자인을 포함하세요.  
> * API 호출은 Next.js의 Server Actions 또는 Supabase Client(클라이언트 컴포넌트용)를 적절히 분리하여 사용하세요.
